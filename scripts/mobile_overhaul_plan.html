UX Overhaul Plan
แผนปรับปรุงระบบทั้งหมดเพื่อประสบการณ์การเล่น Solo D&D AI DM ที่สมบูรณ์แบบบนมือถือ — UX ที่ดีขึ้น UI ที่เหมาะกับมือถือ และฟีเจอร์ใหม่ที่ทำให้เกมสนุกขึ้น

Section 01
สถานะปัจจุบันและปัญหา
ระบบ D&D Solo AI DM ปัจจุบันทำงานได้ครบทั้ง 36 domain modules + 10 engine chapters + AI DM Layer + Open5e integration แต่ UX/UI ยังไม่เหมาะกับการเล่นบนมือถือ และมีปัญหาหลายจุดที่รบกวนประสบการณ์การเล่น จากการตรวจสอบพบปัญหาหลักดังนี้:
0
Responsive CSS

useState ใน 1 component
6
Modals รวมกัน
~30px
Grid cell บนมือถือ (เล็กเกิน)
ปัญหาหลัก 6 จุด
1. Theme ไม่ centralized — CSS อยู่ใน template string ใน DnDSolo.tsx รวม 4,972 บรรทัด แก้ไขยาก
2. ไม่มี responsive CSS — ใช้แค่ max-width:640px ไม่มี breakpoint สำหรับ tablet/desktop
3. ปุ่มเยอะเกิน — 6 modals (Map, Character, Shop, Quest, AI DM, Content) อยู่ใน header row เดียว
4. Combat grid เล็กเกิน — 12×10 grid = ~30px ต่อช่องบนมือถือ ต่ำกว่ามาตรฐาน 44px tap target
5. ไม่มี onboarding — ผู้เล่นใหม่ไม่รู้ว่าต้องทำอะไร ไม่มี tutorial หรือ sample character
6. Shop ไม่มี search — 35+ อาวุธ ไม่มีวิธีกรอง ต้องเลื่อนหาทีละตัว
Key Finding
package.json มี dependencies ครบที่ต้องใช้แล้ว: Zustand 5, framer-motion 12, vaul (Drawer), cmdk (Command), embla-carousel, react-resizable-panels, @dnd-kit, sonner — เราแค่ต้อง wire in สิ่งที่มีอยู่แล้ว เกือบไม่ต้องติดตั้งอะไรเพิ่ม

Section 06
Component Tree ใหม่
แยก DnDSolo.tsx (4,972 บรรทัด) เป็น component tree ที่ชัดเจน:
AppShell ├── SceneHeader (location + time + weather) ├── BottomNav / SidebarNav │ ├── ChatPanel (DM narration + player input) │ ├── CombatPanel (grid + actions + initiative) │ ├── MapPanel (world map + fast travel) │ └── CharacterPanel (stats + skills + spells + items) ├── ContextualQuickActions (เปลี่ยนตาม context) ├── Drawer (shop / quest / AI DM helper / content) └── TurnBanner (framer-motion, combat only)
Section 07
Contextual Quick Actions
ระบบปุ่มลัดที่เปลี่ยนตามสถานการณ์ — ผู้เล่นเห็นแค่ปุ่มที่เกี่ยวข้องใน context ปัจจุบัน ลดความสับสนและเร่งการตัดสินใจ
ContextQuick ActionsTrigger
Townคุย, ร้านค้า, พัก, เดินทาง, เควสต์บอร์ดscene.type = "town"
Combatโจมตี, เวท, ไอเทม, เคลื่อนที่, Dash, Dodge, หนีcombat != null
Dungeonสำรวจ, ฟัง, ซ่อน, เปิดประตู, ตรวจสอบscene.type = "dungeon"
Socialโน้มน้าว, ข่มขู่, หลอก, สังเกต, ค้าขายintent = negotiate/persuade
Wildernessสำรวจ, ติดตามรอย, หาทาง, ตั้งแคมป์, ล่าscene.type = "wilderness"
Restพักสั้น, พักยาว, เฝ้ายาม, ตื่นintent = rest หรือปุ่มพัก
Section 08
Combat UX บนมือถือ
การต่อสู้บนมือถือต้องปรับหลายจุด — grid เล็กเกิน ปุ่มกดยาก ไม่รู้ว่าถึงตาใคร ต้องแก้ทั้งหมด:
Grid Zoom/Pan
ใช้ react-zoom-pan-pinch: pinch ขยาย grid 2-3 เท่า, ลากนิ้วเพื่อ pan ไปมา
Default view: 6×6 ช่องรอบผู้เล่น (เห็นพื้นที่ใกล้ตัวชัดเจน)
Pinch out: ขยายเห็นเต็ม 12×10 กริด
Theater of Mind toggle: ซ่อน grid แสดงแค่ HP/AC/status ของศัตรู
Movement Preview
กดที่พื้นเขียวเพื่อเคลื่อนที่ — แต่ก่อนกด แสดงพื้นเขียวทุกช่องที่เดินถึงได้
ใช้ BFS algorithm คำนวณจาก speed (6 ช่อง = 30 ฟุต) ลบด้วย difficult terrain
AoE preview: เลือกเวทที่มี AoE → แสดงวงกลม/กรวย/เส้นบน grid ก่อนยืนยัน
Initiative Timeline
แสดงเป็นแถบแนวนอนด้านบน: [Goblin 20] → [Goblin 18] → [คุณ 15] → ...
ไฮไลต์ตาปัจจุบัน, แสดง HP bar ของแต่ละตัว
Turn Banner: "ถึงตาคุณ!" (เขียว) / "Goblin กำลังเคลื่อนที่..." (แดง) ด้วย framer-motion

Top Risks
1. Store Migration (P2.1) — 62 useState → Zustand slices อาจทำให้ feature พักชั่วคราว ต้อง migrate ทีละ slice และเทสหลังแต่ละ slice
